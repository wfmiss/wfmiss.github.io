微机原理指令集

# 一、寻址方式

# 立即数（imm）寻址方式：

  • 8位数值（00H-FFH）
  • 16位数值（0000H-FFFFH）
  • 字符常量
  • 只能在源（src）的位置

# 寄存器寻址：

  • 操作数存放在cup的内部寄存器reg中：
    • 8位寄存器r8：AH、AL、BH、BL、CH、CL、DH、DL
    • 16位寄存器r16：AX、BX、CX、DX、SI、DI、SP、BP
    • 4个段寄存器：CS、DS、ES、SS
  • 寄存器名表示其内容（操作数）
  • CS不能在目（dst）位置，只能在源（src）位置

# 存储器寻址：

  • 操作数在主存储器中，用主存有效地址（EA）表示
  • 8086设计了5种存储器寻址方式
    • 1、直接寻址方式
    • 2、寄存器间接寻址方式
    • 3、寄存器相对寻址方式
    • 4、基址变址寻址方式
    • 5、相对基址变址寻址方式

1、直接寻址方式

  • 有效地址（EA）在指令种直接给出；MOV AX,[2000H]
  • 在中括号内表示有效地址（EA）
  • 默认的段地址DS，可用段超越前缀改变；MOV AX,ES:[2000H]

2、寄存器间接寻址方式

  • EA存放在BX、SI、DI或BP中；
  • EA为BX/SI/DI时，默认是DS；
  • BP默认是SS；
  • 可以使用段超越前缀改变

3、寄存器相对寻址方式

  • EA=BX/BP/DI/SI+8/16位位移量；、MOV AX,06H[SI]=MOV AX,[SI+06H]
  • EA为BX/SI/DI时，默认是DS；
  • BP默认是SS；
  • 可以使用段超越前缀改变

4、基址变址寻址方式

  • EA=BX/BP（基址）+SI/DI（变址）、MOV AX,[BX+SI]、MOV AX,[BP][SI]
  • EA为BX/SI/DI时，默认是DS；
  • BP默认是SS；
  • 可以使用段超越前缀改变

5、相对基址变址寻址方式

  • EA=BX/BP（基址）+SI/DI（变址）+8/16位位移量、MOV AX,[BX+SI+6]、MOV AX,6[BP+DI]
  • EA为BX/SI/DI时，默认是DS；
  • BP默认是SS；
  • 可以使用段超越前缀改变

# 二、数据传送类指令

除标志寄存器传送指令外，均不影响标志位。

重点：MOV、XCHG、PUSH、POP、LEA

了解：XLAT、LDS、LES

1、传送指令：MOV 目(dst),源(src)

  • 立即数传送：
    • 立即数一定在src位置；
    • 源长度与目的操作数需长度一致
  • 寄存器传送：
    • 速度块
    • 目与源需长度一致
  • 存储器传送：
    • 不存在存储器向存储器的传送指令
    • 目和源不能同时是存储器寻址
  • 段寄存器传送

2、交换指令：XCHG reg目(dst),reg/mem源(src)

  • 目与源的值交换
  • 双操作数长度一致
  • 立即数不可以使用

3、换码指令：XLAT（了解）

  • AL<—DS:[BX+AL]
  • 将BX指定的缓冲区中，AL指定的位移处的一个字节数取出赋给AL
  • 隐含寻址（零地址指令，不带操作数）

4、 取有效地址：LEA r16目(dst),mem源(src)

  • 取源的有效地址给目操作数

# 堆栈操作指令

  • 堆栈占用主存区域;
  • 采用“先进后出”或“后进先出”原则；
  • 位于堆栈段中，段地址保存在SS段寄存器；
  • 堆栈只有一个出口，栈顶；
  • 用堆栈指针寄存器SP指定栈顶位置；
  • 堆栈只有两种基本操作：进栈PUSH和出栈POP。

进栈指令：PUSH r16/m16/seg

  • SP<—SP-2
  • SS:[SP]<—r16/m16/seg
  • 字操作

出栈指令：POP r16/m16/seg

  • r16/m16/seg<—SS:[SP]
  • SP<—SP+2
  • 字操作

# 标志操作指令

1、标志位操作指令（串操作）

  • CLC ;复位进位标志位：CF=0
  • STC ;置位进位标志位：CF=1
  • CMC ;求反进位标志位：CF=-CF
  • CLD ;复位方向标志位：DF=0
  • STD ;置位方向标志位：DF=1
  • CLI ;复位中断标志位：IF=0（关中断）
  • STI ;置位中断标志位：IF=1（开中断）

2、标志寄存器低字节传送AH的指令：LAHF

3、标志寄存器出入堆栈指令：PUSHF、POPF

# 三、算数运算类

# 加减乘除

运算结果对状态标志位影响

加法：ADD、ADC、INC

减法：SUB、SBB、DEC、CMP、NEG(求补)

1、加和减指令

ADD 目(dst),源(src) ;dst<—dst+src

SUB 目(dst),源(src) ;dst<—dst-src

2、带进位的加和减指令

ADC 目(dst),源(src) ;dst<—dst+src+CF

SBB 目(dst),源(src) ;dst<—dst-src-CF

3、比较指令CMP（compare）

CMP 目(dst),源(src) ;dst-src

  • 差值不回送目的操作数
  • 影响状态标志

4、增量和减量指令

INC reg/mem

DEC reg/mem

  • 均为单操作数指令
  • 不影响标志位CF

5、求补指令

NEG reg/mem ;reg/mem<—0-reg/mem

  • 单操作数指令
  • 不能用立即数
  • 影响标志位

# 符号扩展指令

CBW ;AL符号扩展成AX

CWD ;AX符号扩展成DX

  • 零地址指令
  • 隐含的
    • CBW：AL—>AX
    • CWD：AX—>DX,AX(双字)

# 乘法和除法指令

# 乘法指令

  • 分无符号和有符号指令
  • 单操作数指令（隐含使用AL和AX）

MUL opr ;无符号指令

IMUL opr ;有符号指令

格式：

AL*opr（字节）—> AX

AX*opr（字）—>DX,AX

  • opr不能是立即数，可以是reg/mem
  • 隐含被乘数
  • 对标志位影响
    • 字节*字节=字节：CF=OF=0
    • 字节*字节=字 ：CF=OF=1
    • 字 *字 =字 ：CF=OF=0
    • 字 *字 =双字：CF=OF=1
  • 其他标志位无定义

# 除法指令

  • 分无符号和有符号指令
  • 单操作数指令（被除数隐含使用AX和DX,AX）

DIV opr ;无符号指令

IDIV opr ;有符号指令

  • opr不能是立即数，可以是reg/mem
  • 除数为0或商出范围，OF=1溢出
  • 对其他标志位无定义

AX/opr(字节)—>商(AL),余(AH)

DX,AX/opr(字)—>商(AX),余(DX)

# 四、位操作指令

以二进制进行数据的操作

注意：对标志位的影响

# 1、逻辑运算指令

AND、OR、XOR、NOT、TEST

  • 设置：CF=OF=0
  • 结果影响SF、ZF和PF状态，对AF未定义
  • 单操作数：NOT 不影响任何标志位

逻辑与：AND（有0出0）

AND 目(dst),源(src) ;dst<—dst与src

  • 清零
    • AND AX,0
    • AND AL,0FH
    • AND AX,AX

逻辑或：OR（有1出1）

OR 目(dst),源(src) ;dst<—dst 或src

  • 置位：
    • OR AX,AX ;AX不变
    • OR AL,OFH ；

逻辑异或：XOR（相同为0，不同为1）

XOR 目(dst),源(src) ;dst<—dst异或src

  • 取反、清零：
    • XOR AX,AX
    • XOR AL,03H

测试指令：TEST

TEST 目(dst),源(src) ;dst与src

  • 不保留结果
  • test AL,01H ;测试D0是否为1。
  • 测试某一位

取反指令：NOT

NOT reg/mem

  • 取反
  • 单操作数指令
  • 不影响任何标志位

# 2、移位指令

无符号（逻辑）：SHL、SHR；有符号（算数）：SAL、SAR

L：左，R：右

  • 移出的位进入CF
  • 移位后的结果影响SF、ZF、PF

逻辑左移指令：SHL

SHL reg/mem,1/CL ;最低位补0，最高位进入CF

逻辑右移指令：SHR

SHR reg/mem,1/CL ;最高位补0，最低位进入CF

算数左移指令：SAL

SAL reg/mem,1/CL ;最低位补0，最高位进入CF

算数右移指令：SAR

SAR reg/mem,1/CL ;最高位不变，最低位进入CF

# 3、循环移位指令

不带进位：ROL、ROR；带进位：RCL、RCR

  • 设置进位标志CF，但不影响SF、ZF、PF、AF标志

循环左移：ROL

ROL reg/mem,1/CL ;最高位移入CF和最低位

循环左右移：ROR

ROR reg/mem,1/CL ;最低位移入CF和最高位

进位循环左移：RCL

RCL reg/mem,1/CL ;先将CF的值移入最低位，再将最高位的值移入CF

进位循环右移：RCL

RCR reg/mem,1/CL ;先将CF的值移入最高位，再将最低位的值移入CF

# 1、传送：从源存储区到目的区

  • MOVSB

意思是搬移一个字节，它是把 DS:SI 所指地址的一个字节搬移到 ES:DI 所指的地址上，搬移后原来的内容不变，但是原来 ES:DI 所指的内容会被覆盖而且在搬移之后 SI 和 DI 会自动地指向下一个要搬移的地址。

  • MOVSW

MOVSB、MOVSW 和 MOVSD 指令将数据从 ESI 指向的内存位置复制到 EDI 指向的内存位置。（根据方向标志位的值）这两个寄存器自动地增加或减少：

MOVSB 传送（复制）字节
MOVSW 传送（复制）字
MOVSD 传送（复制）双字

MOVSB、MOVSW 和 MOVSD 可以使用重复前缀。方向标志位决定 ESI 和 EDI 是否增加或减少。增加 / 减少的量如下表所示：

指令 ESI 和 EDI 增加或减少的数值
MOVSB 1
MOVSW 2
MOVSD 4

# 2、存入串：将AX的内容存入目的存储区

  • STOSB

该指令为单字符输出指令，调用该指令后，可以将累加器AL中的值传递到当前ES段的DI地址处，并且根据DF的值来影响DI的值，如果DF为0，则调用该指令后，DI自增1。

  • STOSW

# 3、从串取：从源存储区的内容送AX

  • LODSB
  • LODSW

串操作指令LODSB/LODSW是块读出指令，其具体操作是把SI指向的存储单元读入累加器,其中LODSB是读入AL,LODSW是读入AX中,然后SI自动增加或减小1或2位.当方向标志位DF=0时，则SI自动增加；DF=1时，SI自动减小。

与LODSB/LODSW类似的，STOSB/STOSW是块写入指令，其具体操作是把累加器的内容写入到指向的存储单元中。其中STOSB是从AL中读入,STOSW是从AX中读入,然后DI自动增加或减小1或2位.当方向标志位DF=0时，则DI自动增加；DF=1时，DI自动减小。

# 6、三种控制次数的指令（配合上面指令使用）

  • REP

    • CX<—CX-1

    • 判断CX=0？若CX=0，退出

    • REP STOS m8 ：使用 AL 填写位于 ES:[(E)DI] 的 (E)CX 个字节

  • REPE/REPZ ；找不相等的字符
    • CX<—CX-1
    • 判断CX=0或ZF=0，则退出
    • REPE CMPS m8,m8 ：在 ES:[(E)DI] 与 DS:[(E)SI] 中查找不匹配的字节
  • 串重复前缀REPNE/REPNZ；找相等的字符
    • CX<—CX-1
    • 判断CX=0或ZF=1，则退出
    • REPNE CMPS m8,m8 ：在 ES:[(E)DI] 与 DS:[(E)SI] 中查找匹配字节

# 九、伪指令

  • 不参与程序执行，起到指示说明的作用

类型名 PTR 名字/标号：当无法通过任一个操作数去确定操作类型时，需要显示指明长度时使用。

  • Byte PTR、Word PTR等；

OFFSET 名字/标号：返回EA（偏移地址地址）

SEG 名字/标号：返回段地址

END：表示源程序到此结束

EQU：符号定义伪指令

  • EQU和“=”
  • 符号名 EQU 常数/数值表达式
  • 符号名 = 常数/数值表达式
  • EQU定义不占内存
  • EQU不能重复定义符号名，但“=”允许有重复赋值

# 变量的定义

汇编语言指令格式：

变量名 伪指令 初值表

变量名：表示初值表首元素的逻辑地址；

伪指令：DB(字节8位)、DW(字16位)、DD(双字32位)

初值表：用逗号分隔的参数

复制操作符DUP定义多个相同元素：

格式：重复次数 DUP (重复参数1、重复参数2、……)

DB：

  • 无符号数：0——255
  • 有符号数：-128——+127
  • 字符串常数

变量的定位：

  • 段中定义的第一变量从0地址开始；
  • 按照定义的先后顺序依次分配存储空间；
  • ORG控制数据或代码所在的偏移地址：ORG 参数

名字和标号的属性：

  • 名字指向一条伪指令，标号指向一条硬指令。
  • 具有两类属性：
    • 地址：段地址和偏移地址；
    • 类型：
      • 变量类型：BYTE、WORD和DWORD；
      • 标号、段名、子程序名的类型：NEAR(近)和FAR(远)

# 十、分支程序设计

  • 根据条件是真或假决定执行与否
  • 判断的条件指令，如cmp，test，执行后形成的状态标志

# 十一、循环程序设计

3个组成部分：

  • 循环初始部分：为开始循环准备必要的条件，如循环次数、循环体需要的数值等。
  • 循环体部分：指重复执行的程序部分，其中包括对循环条件等的修改程序段
  • 循环控制部分：判断循环条件是否成立，决定是否继续循环
